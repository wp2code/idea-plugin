# Controller接口权限SQL生成插件 开发文档

## 第一章 项目概述
### 1.1 项目背景
在Spring Boot/Spring Cloud项目的权限系统开发中，接口权限入库需要手动编写大量INSERT SQL语句，效率低、易出错，且重复工作量大。同时多语言架构的系统需要权限资源支持国际化名称配置，不同接口的国际化标识存在差异化，无需模板统一预设。
本插件通过解析Controller代码自动提取接口元数据，配合可配置、可复用的SQL模板，批量生成标准化的权限入库SQL；国际化标识采用接口级独立手动配置、允许为空的设计，灵活适配多语言系统的差异化需求，大幅提升开发效率与通用性。

### 1.2 项目目标
1. 自动解析Controller类/单个接口方法的路由、请求方式、注释等元数据
2. 支持SQL模板可视化配置、本地持久化、导入导出，实现多环境共享
3. 支持资源类型、权限类型字典化统一管理，可灵活扩展
4. **国际化标识采用接口级纯手动配置，无模板默认值，支持清空为空，模板切换不影响接口配置**
5. 支持模板扩展字段自定义配置，键值对结构灵活扩展，适配各类业务差异化字段需求
6. 批量生成权限INSERT SQL，提供预览、一键复制、导出文件三种输出方式
7. 资源编码支持手动录入、HTTP接口拉取双模式，满足不同业务场景

### 1.3 适用场景
Java Spring Boot/Spring Cloud项目的权限系统开发，用于批量生成菜单、接口、按钮、页面等类型权限的入库SQL脚本，尤其适配多差异化字段、多语言国际化架构的权限系统。

## 第二章 整体架构设计
### 2.1 模块划分
插件采用分层架构，共分为6个核心模块：

| 模块 | 职责说明 |
| ---- | ---- |
| 代码解析模块 | 解析Java文件，提取Controller接口的路由、请求方式、注释、国际化注解等元数据 |
| UI交互模块 | 侧边面板、管理弹窗、提示弹窗等所有前端交互组件的渲染与事件处理，含扩展字段动态表单、接口级国际化输入控件 |
| 模板管理模块 | 模板的新增、编辑、删除、导入、导出全生命周期逻辑处理，含扩展字段管理 |
| 字典管理模块 | 资源类型、权限类型字典数据的增删改查与下拉数据源维护 |
| SQL生成模块 | 模板变量解析、批量替换、SQL拼接、输出处理，支持国际化变量、扩展字段变量渲染，兼容空值处理 |
| 数据持久层 | SQLite数据库操作封装，提供统一的CRUD接口，支持扩展字段JSON存取 |

### 2.2 技术选型建议
- UI框架：IDEA Plugin SDK + Swing/JavaFX
- 本地数据库：SQLite（嵌入式，无需额外服务）
- 代码解析：PSI（Program Structure Interface，IDEA原生代码解析API）
- JSON处理：Jackson/Gson，用于扩展字段序列化与反序列化
- IDE 兼容：IntelliJ IDEA 2021.3.2+

## 第三章 功能详细设计
### 3.1 代码解析模块
#### 3.1.1 触发方式
编辑器右键菜单新增选项：**「解析接口生成权限SQL」**，支持两种选中范围：
1. 选中单个接口方法：仅解析当前方法
2. 选中Controller类（类名/整个Java文件）：解析该类下所有接口方法

#### 3.1.2 解析规则
1. **路由拼接**：提取类上`@RequestMapping`的父路径 + 方法上`@GetMapping/@PostMapping/@PutMapping/@DeleteMapping`等注解的子路径，拼接成完整路由
2. **请求方式**：根据注解提取GET/POST/PUT/DELETE等请求方法
3. **接口名称**：优先提取方法注释、`@ApiOperation`注解的value值，无注释则为空
4. **国际化标识提取（可选扩展）**：可识别`@I18nCode`等自定义注解，自动填充对应接口的国际化标识；无注解时默认为空，用户可手动输入、修改或清空

#### 3.1.3 输出数据结构
每个接口解析后生成统一对象：
```java
class InterfaceInfo {
    private String requestMethod;    // 请求方式
    private String routePath;        // 完整路由地址
    private String interfaceName;    // 接口名称/注释
    private String i18nResourceCode; // 接口国际化标识，手动录入，允许为空
}
```

#### 3.1.4 异常处理
- 选中非Controller类/非接口方法：弹窗提示「请选中有效的Controller类或接口方法」
- 代码语法错误/解析失败：弹窗提示「接口解析失败，请检查代码格式」，不中断插件运行

### 3.2 模板管理模块
#### 3.2.1 模板业务字段
对应数据库`script_template`表，核心业务字段：

| 字段名 | 说明 | 约束 |
| ---- | ---- | ---- |
| templateName | 模板名称 | 非空、全局唯一 |
| templateContent | SQL模板内容 | 非空 |
| creatorIdValue | 默认创建人ID | - |
| updaterIdValue | 默认更新人ID | - |
| httpCodeApi | 资源编码HTTP拉取接口地址 | - |
| defaultCodeMode | 资源编码默认模式 | 0-手动录入，1-HTTP获取，默认0 |
| extConfig | 扩展字段配置 | JSON格式键值对，存储自定义业务字段，可空 |

> 国际化说明：模板不再存储国际化标识默认值，国际化标识为接口级独立配置，由用户手动录入，支持为空。

#### 3.2.2 模板新增
- 入口：侧边面板「创建脚本模板」按钮
- 前置校验：模板名称非空、全局唯一；扩展字段key不可重复、不可含特殊字符
- 业务逻辑：表单数据（含扩展字段JSON）入库，刷新模板下拉列表，自动选中新增的模板

#### 3.2.3 模板编辑
- 入口：选中模板后点击「编辑模板」按钮
- 状态切换：点击编辑后，「创建脚本模板」按钮文字切换为「保存修改」
- 回填逻辑：将选中模板的所有业务字段（含扩展字段键值对）回填到表单
- 保存校验：模板名称非空、排除自身后全局唯一；扩展字段key合法性校验
- 未保存拦截：编辑未保存时切换模板/导入模板，弹出二次确认「当前修改未保存，确认丢弃？」

#### 3.2.4 模板删除
- 入口：选中模板后点击「删除模板」按钮
- 二次确认：弹出弹窗「确定删除模板【{模板名称}】？删除后无法恢复」
- 执行逻辑：确认后删除数据库对应记录，清空表单选中状态，刷新下拉列表

#### 3.2.5 模板导出
- 入口：选中模板后点击「导出模板」按钮
- 文件规则：默认文件名`{模板名称}_template.json`，UTF-8编码，标准JSON格式
- 导出字段：仅导出业务字段，不含自增ID、创建/更新时间等系统字段，包含extConfig扩展字段
- 交互：调用系统文件保存对话框，支持用户自定义保存路径和文件名

#### 3.2.6 模板导入
- 入口：「导入模板」按钮（始终可用）
- 文件过滤：仅支持`.json`格式文件
- 校验流程（按顺序执行，任一不通过终止并提示）：
  1. 文件大小校验：≤1MB，超出提示「文件过大，请选择1MB以内的JSON文件」
  2. 格式校验：JSON格式合法性，错误提示「模板文件格式错误，请选择合法的JSON文件」
  3. 字段校验：必填字段完整性，缺失提示「模板文件缺少必填字段：xxx」；扩展字段为选填字段，允许为空
  4. 扩展字段校验：扩展字段需为标准键值对结构，key命名需符合规范（英文、数字、下划线）
  5. 重名校验：模板名称全局唯一，重复提示「模板名称已存在，请修改后重新导入」
- 校验通过：数据（含扩展字段）入库，刷新下拉列表，自动选中导入的模板并回填表单

#### 3.2.7 模板切换联动规则
切换模板下拉选项时：
1. 同步更新表单中模板全局配置字段：创建人ID、更新人ID、HTTP拉取地址、默认编码模式、扩展字段键值对
2. **接口列表配置保留规则**：
  - 接口的国际化标识完全保留，不受模板切换影响
  - 用户手动修改过的资源编码、资源类型、权限类型等字段，全部保留原值，不随模板切换覆盖
  - 未进行过任何手动编辑的接口，自动同步模板的默认编码模式
3. 新增解析的接口，国际化标识默认为空，用户可随时手动输入、修改、清空

#### 3.2.8 扩展字段交互规则
1. 模板配置区提供「扩展字段」配置区域，默认折叠，可展开编辑
2. 支持动态添加、删除键值对行，每行包含「字段Key」「字段Value」两个输入框
3. 字段Key实时校验：不可为空、不可重复、仅支持英文/数字/下划线，不符合时标红提示
4. 扩展字段为模板级全局配置，所有生成的SQL共用同一套扩展字段值

#### 3.2.9 模板导出JSON示例
```json
{
  "templateName": "标准权限SQL模板（扩展字段）",
  "templateContent": "INSERT INTO sys_permission (resource_address, resource_name, i18n_resource_code, resource_code, resource_type, permission_type, remark, system_code, sort, create_time, update_time, creator_id, updater_id, deleted) VALUES ('${resourceAddress}', '${resourceName}', '${i18nResourceCode}', '${resourceCode}', ${resourceType}, ${permissionType}, '${remark}', '${ext.systemCode}', ${ext.sort}, DATETIME('now'), DATETIME('now'), ${creatorId}, ${updaterId}, 0);",
  "creatorIdValue": "1",
  "updaterIdValue": "1",
  "httpCodeApi": "http://127.0.0.1:8080/api/permission/getCode",
  "defaultCodeMode": 1,
  "extConfig": {
    "systemCode": "admin-system",
    "sort": "10"
  }
}
```

### 3.3 字典管理模块
#### 3.3.1 字典分类
两类独立字典，分表存储：
1. 资源类型：菜单、接口、按钮、页面、消息topic
2. 权限类型：查看、新增、编辑、删除、控制

#### 3.3.2 入口
接口列表中，「资源类型」「权限类型」下拉框右侧的「管理」小按钮，点击打开对应字典管理弹窗。

#### 3.3.3 弹窗布局
- 顶部：「新增」按钮
- 中间：字典列表表格，列：类型编码、类型名称、排序、操作列（编辑/删除）
- 底部：「关闭」按钮

#### 3.3.4 新增字典项
- 点击「新增」展开表单，字段：类型编码（数字）、类型名称（文本）、排序（数字，默认0）
- 校验规则：编码非空、全局唯一；名称非空
- 提交后入库，列表实时刷新

#### 3.3.5 编辑字典项
- 点击「编辑」回填当前行数据
- 校验规则：编码非空、排除自身后全局唯一；名称非空
- 提交后更新数据库，列表实时刷新

#### 3.3.6 删除字典项
- 点击「删除」弹出二次确认「确定删除该字典项？删除后不可恢复」
- 确认后删除数据库对应记录，列表刷新
- 约束：删除不校验业务数据引用，已选中该类型的接口保留原值，下拉选项不再展示该项

#### 3.3.7 下拉数据源规则
所有页面的资源类型、权限类型下拉框，均实时读取对应字典表数据，按`sort`字段升序排列。

### 3.4 接口配置模块
#### 3.4.1 接口列表布局
- 顶部操作栏：全选/取消全选复选框
- 每行对应一个解析后的接口，包含以下配置项：
  1. 勾选框：控制当前接口是否参与SQL生成
  2. 接口标识：请求方式标签 + 完整路由地址
  3. 资源名称：单行输入框，默认填充接口注释，支持编辑、允许为空
  4. **国际化标识**：单行输入框
     - 初始值：优先读取代码解析的国际化注解，无注解则默认为空
     - 编辑规则：完全手动配置，可自由输入、修改、清空为空，无强制必填校验
     - 独立性：不受模板切换影响，每个接口独立配置
  5. 资源编码模块：双模式切换组件
  6. 资源类型：下拉选择框
  7. 权限类型：下拉选择框
  8. 备注：多行文本框，允许为空

> 扩展字段为模板级全局配置，不在接口行单独展示；若需接口级扩展字段，可基于现有架构后续迭代扩展。

#### 3.4.2 资源编码双模式
- **模式切换**：按钮切换「手动录入 / HTTP获取」
- **手动模式**：展示单行输入框，用户手动填写编码
- **HTTP模式**：
  - 地址输入框默认填充模板配置的`http_code_api`，支持手动修改
  - 「拉取编码」按钮：点击发起HTTP GET请求
  - 成功逻辑：解析响应提取编码值，自动回填到输入框
  - 失败逻辑：弹窗提示「编码拉取失败：{具体错误原因}」

#### 3.4.3 编码唯一性校验
1. **实时校验**：手动输入、HTTP回填后，自动校验所有勾选接口的资源编码是否重复；重复时对应输入框标红，悬浮提示「资源编码重复」
2. **生成前强校验**：点击「生成脚本」时执行全局校验，存在重复编码则弹窗拦截，不生成SQL
> 注：国际化标识无唯一性校验要求，允许重复、允许为空。

#### 3.4.4 批量操作
- 全选/取消全选：一键切换所有接口的勾选状态
- 批量设置国际化标识：可扩展支持选中多个接口后批量填充统一国际化标识、批量清空

### 3.5 SQL生成与输出模块
#### 3.5.1 模板变量规则
模板中支持四类变量，生成时自动替换：
1. 接口动态变量：每条接口独立替换，取值自接口表单
2. 全局模板变量：所有接口共用，取自模板配置
3. 扩展字段变量：所有接口共用，取自模板扩展字段配置
4. 内置常量：固定值，无需页面配置
   （详细变量规范见第五章）

#### 3.5.2 生成逻辑
- 触发：点击「生成脚本」按钮
- 前置校验：资源编码唯一性校验
- 核心逻辑：遍历所有勾选接口，按顺序替换模板中所有变量，拼接成完整SQL脚本
- **空值处理规则**：
  - 接口国际化标识为空时，`${i18nResourceCode}`变量替换为空字符串（`''`）
  - 其他接口字段（资源名称、备注等）为空时，同样替换为空字符串
  - 扩展字段key不存在时，替换为空字符串
- 变量缺失警告：若模板引用了不存在的扩展字段key，在预览区顶部给出警告提示「存在未定义的扩展字段变量：xxx」
- 输出：将最终SQL写入底部预览文本框

#### 3.5.3 SQL预览区
- 只读多行文本框，支持滚动查看，可选配置SQL语法高亮
- 支持变量警告提示展示
- 预览区为空时，「一键复制全部SQL」「导出SQL文件」按钮置灰不可点击

#### 3.5.4 一键复制
- 点击「一键复制全部SQL」按钮
- 将预览区全部内容复制到系统剪贴板
- 成功后右下角Toast提示「复制成功」，2秒自动消失

#### 3.5.5 导出SQL文件
- 点击「导出SQL文件」按钮
- 调用系统文件保存对话框，默认文件名`permission_sql_yyyyMMddHHmmss.sql`
- 文件内容：预览区全部SQL脚本，UTF-8纯文本编码
- 保存成功后Toast提示「SQL文件导出成功」

## 第四章 数据库设计
### 4.1 数据库选型
SQLite本地嵌入式数据库，随插件安装自动初始化，无需额外部署服务。

### 4.2 初始化规则
插件首次启动时，自动执行建表语句，并插入字典初始数据。版本升级时自动执行表结构增量更新。

### 4.3 详细表设计
#### 4.3.1 资源类型字典表 `dict_resource_type`
| 字段名 | 类型 | 约束 | 说明 |
| ---- | ---- | ---- | ---- |
| id | INTEGER | 主键、自增 | 主键ID |
| type_code | TINYINT | 非空、唯一 | 类型编码 |
| type_name | VARCHAR | 非空 | 类型名称 |
| sort | INTEGER | 默认0 | 排序值，越小越靠前 |
| create_time | DATETIME | 默认CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | 默认CURRENT_TIMESTAMP | 更新时间 |

**建表与初始化SQL**
```sql
CREATE TABLE dict_resource_type (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    type_code TINYINT NOT NULL UNIQUE,
    type_name VARCHAR NOT NULL,
    sort INTEGER DEFAULT 0,
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 初始数据
INSERT INTO dict_resource_type(type_code,type_name,sort) VALUES
(0,'菜单',1),
(1,'接口',2),
(2,'按钮',3),
(3,'页面',4),
(4,'消息topic',5);

-- 更新时间触发器
CREATE TRIGGER trg_dict_res_update AFTER UPDATE ON dict_resource_type
FOR EACH ROW
BEGIN
    UPDATE dict_resource_type SET update_time = CURRENT_TIMESTAMP WHERE id = OLD.id;
END;
```

#### 4.3.2 权限类型字典表 `dict_permission_type`
| 字段名 | 类型 | 约束 | 说明 |
| ---- | ---- | ---- | ---- |
| id | INTEGER | 主键、自增 | 主键ID |
| type_code | TINYINT | 非空、唯一 | 权限编码 |
| type_name | VARCHAR | 非空 | 权限名称 |
| sort | INTEGER | 默认0 | 排序值，越小越靠前 |
| create_time | DATETIME | 默认CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | 默认CURRENT_TIMESTAMP | 更新时间 |

**建表与初始化SQL**
```sql
CREATE TABLE dict_permission_type (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    type_code TINYINT NOT NULL UNIQUE,
    type_name VARCHAR NOT NULL,
    sort INTEGER DEFAULT 0,
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 初始数据
INSERT INTO dict_permission_type(type_code,type_name,sort) VALUES
(0,'查看',1),
(1,'新增',2),
(2,'编辑',3),
(3,'删除',4),
(4,'控制',5);

-- 更新时间触发器
CREATE TRIGGER trg_dict_perm_update AFTER UPDATE ON dict_permission_type
FOR EACH ROW
BEGIN
    UPDATE dict_permission_type SET update_time = CURRENT_TIMESTAMP WHERE id = OLD.id;
END;
```

## 第六章 交互与UI规范
### 6.1 按钮状态规范
- 置灰规则：前置条件不满足时（如未选中模板、预览区为空），对应按钮置灰不可点击
- 按钮分级：
  - 主按钮：生成脚本、保存修改，使用主题色
  - 次按钮：编辑、删除、导入导出，使用默认样式
  - 危险按钮：删除操作，hover时展示红色警示样式

### 6.2 校验提示规范
- 实时校验：输入框标红 + 悬浮tooltip提示错误原因
- 提交校验：模态弹窗展示具体错误信息，带确认按钮
- 成功提示：右下角轻量Toast，2秒自动消失，不阻塞操作
- 警告提示：预览区顶部浅色背景提示条，展示变量缺失等非阻断性警告

### 6.3 弹窗规范
- 二次确认弹窗：标题 + 提示文案 + 取消/确认按钮，不支持点击遮罩关闭，防止误操作
- 表单弹窗：标题 + 表单区域 + 取消/确认按钮，支持点击遮罩关闭
- 管理弹窗：可拖拽、可调整大小，点击关闭按钮直接关闭

### 6.4 扩展字段UI规范
- 配置区域默认折叠，标题栏展示「扩展字段配置（可选）」，点击展开/收起
- 每行两个输入框：左为字段Key（占比40%），右为字段Value（占比50%），末尾为删除按钮
- 底部提供「+ 添加字段」按钮，点击新增一行空白键值对
- 字段Key输入框实时校验命名规范，不符合时标红并tooltip提示

## 第七章 异常处理规范
### 7.1 代码解析异常
- 非Java文件/非Controller类：友好提示，不崩溃
- 代码语法错误：提示「接口解析失败，请检查代码语法」，不中断插件运行
- 空文件/无接口：提示「当前选中类未解析到有效接口」

### 7.2 数据库操作异常
- 增删改查失败：弹窗提示「操作失败：{具体错误信息}」
- 数据库初始化失败：插件菜单置灰禁用，提示「插件初始化失败，请重启IDE」
- 版本升级表结构更新失败：提示「插件数据升级失败，请检查本地数据库文件」
- 扩展字段JSON序列化/反序列化失败：提示「扩展字段格式错误，请检查配置」

### 7.3 HTTP请求异常
- 网络不通、请求超时：提示「网络请求失败，请检查地址和网络」
- 接口返回错误：提示「编码拉取失败，接口返回：{错误信息}」
- 响应解析失败：提示「响应格式解析失败，请检查接口返回格式」

### 7.4 文件操作异常
- 导入文件损坏/格式错误：提示具体错误原因
- 导出无写入权限：提示「文件保存失败，无目录写入权限，请更换路径」
- 文件过大：按预设阈值拦截并提示
- 导入扩展字段格式非法：提示「扩展字段格式错误，请使用标准键值对JSON结构」

### 7.5 模板变量解析异常
- 引用不存在的扩展字段：替换为空字符串，同时给出警告提示
- 变量语法错误：生成SQL时保留原变量字符串，给出警告提示，不阻断生成

## 第八章 非功能需求
### 8.1 性能要求
- 解析含100个接口的Controller，耗时≤1s
- 生成100条SQL脚本，耗时≤0.5s
- 模板下拉、字典下拉加载无明显卡顿（≤200ms）
- 扩展字段增删操作无卡顿，单模板支持最多20个扩展字段

### 8.2 兼容性
- 支持IntelliJ IDEA 2020.3及以上版本
- 支持Spring Boot 2.x、3.x项目
- 兼容Windows、macOS、Linux主流操作系统
- 向下兼容历史版本模板数据，升级后历史模板可正常使用

### 8.3 数据安全
- 所有模板、字典配置仅存储在本地SQLite数据库，不上传任何服务器
- 导入导出文件仅在用户本地操作，无网络传输
- 不收集任何用户代码、项目信息
